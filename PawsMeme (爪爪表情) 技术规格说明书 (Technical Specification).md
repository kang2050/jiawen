# PawsMeme (爪爪表情) 技术规格说明书 (Technical Specification)

## 1. 项目概述
PawsMeme 是一款面向苹果生态（iOS/macOS）及微信小程序的轻量化宠物 AIGC 应用。其核心功能是通过引导式拍摄采集宠物特征，利用云端 LoRA 训练生成个性化的宠物表情包。

---

## 2. 系统架构设计

系统采用 **“端侧采集与交互 + 云端训练与推理”** 的混合架构：

| 组件 | 平台 | 核心职责 |
| :--- | :--- | :--- |
| **PawsMeme iOS/macOS** | Swift / SwiftUI | 实时动物姿态检测、骨架引导拍摄、数据加密上传、表情包展示。 |
| **PawsMeme 小程序** | WXML / JavaScript | 轻量化数据采集入口、表情包社交分享、生成进度查看。 |
| **云端算力中心 (Backend)** | Python / GPU Cluster | 图像预处理、LoRA 模型训练 (FLUX/SDXL)、表情包生成推理、API 接口。 |

---

## 3. 关键技术模块实现

### 3.1 骨架引导拍摄 (Skeleton-Guided Capture)
这是确保“像我家孩子”的关键。通过 iOS 原生的 **Vision Framework** 实现。

*   **技术路径**：
    *   使用 `VNDetectAnimalBodyPoseRequest` 实时识别宠物的 25 个核心关键点（耳朵、眼睛、口鼻、关节、尾巴）。
    *   **引导逻辑**：在相机界面叠加一个半透明的“目标骨架”模板。
    *   **自动快门**：当实时检测到的宠物骨架与目标模板重合度（Cosine Similarity）超过 85% 时，App 自动触发高画质拍摄。
*   **采集需求**：引导用户完成 5 组动作（正坐、侧卧、张嘴、仰头、歪头），每组自动抓拍 3-5 张，共计 20 张高质量训练素材。

### 3.2 云端 LoRA 训练流水线
采集到的数据将通过加密通道上传至云端进行快速微调。

*   **模型选型**：推荐使用 **FLUX.1 [dev]** 或 **SDXL** 作为底模，因为它们对动物特征的理解和文本遵从度极高。
*   **训练流程**：
    1.  **自动打标 (Auto-Captioning)**：利用 BLIP-2 或 LLaVA 对上传照片进行描述（例如：*a photo of [PET_NAME] the golden retriever, looking at camera*）。
    2.  **快速训练**：使用 `fal.ai` 或 `replicate` 的服务器级 API 进行 LoRA 微调，通常在 3-5 分钟内完成。
    3.  **身份锁定**：在训练中加入特定的触发词（如 `oh_my_dog`），确保生成的表情包能精准召唤出该宠物的特征。

### 3.3 表情包生成逻辑
*   **Prompt 模板库**：预设 50+ 个社交热梗模板（如“打工人”、“不想起床”、“暗中观察”）。
*   **推理参数**：使用 LoRA 权重配合 **ControlNet (Canny/Depth)**。ControlNet 用于锁定表情包模板的构图，LoRA 用于替换其中的宠物形象。

## 4. 多端开发指南

### 4.1 iOS / macOS (Swift)
*   **UI 框架**：使用 SwiftUI 构建响应式界面，确保 macOS（通过 Catalyst 或原生）和 iOS 体验一致。
*   **核心库**：
    *   `Vision`: 用于 `VNDetectAnimalBodyPoseRequest`。
    *   `AVFoundation`: 用于自定义相机控制。
    *   `Combine`: 用于处理云端训练进度的异步状态轮询。

### 4.2 微信小程序
*   **采集方案**：受限于小程序性能，无法进行复杂的实时骨架比对。建议采用“静态框引导”方案，即在屏幕上画出猫/狗的轮廓虚线，提示用户对齐。
*   **上传**：利用 `wx.uploadFile` 将照片流式传输至后端服务器。

---

## 5. MVP (最小可行性产品) 开发路径

1.  **Phase 1 (2周)**：搭建基础 iOS 相机 Demo，跑通 Vision 框架的动物骨架检测，实现引导框交互。
2.  **Phase 2 (1周)**：对接 `fal.ai` 或 `replicate` 的 API，实现“上传照片 -> 云端训练 -> 获取模型”的闭环。
3.  **Phase 3 (1周)**：开发表情包生成界面，内置 5 个基础模板，实现一键生成并保存到相册。

---

## 6. 技术难点与应对
*   **异形宠物识别**：Vision 框架目前对猫狗支持最好。对于特种宠物（如兔子、仓鼠），需要降级为普通的轮廓引导。
*   **算力成本控制**：初期建议直接调用成熟的 Serverless GPU 平台（按量计费），避免自建服务器。
